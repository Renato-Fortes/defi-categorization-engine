Project name: DeFi Categorization Engine + Accountant Console (Demo)

Goal (what to build):
Build a modern, user-friendly web app that demonstrates a “DeFi Categorization Engine” for crypto accountants. The app should import a CSV export from tools like Koinly/Cryptio (or a sample CSV), automatically re-label common DeFi transactions (starting with Aave Borrow/Repay) using on-chain decoding + deterministic rules, show a review UI (with confidence + explanation), allow manual overrides and bulk actions, and export a corrected CSV.

This is a demo MVP to show a first customer (crypto accountant) within hours, so focus on end-to-end workflow and “wow moment” accuracy for a narrow set of transactions rather than broad protocol support.

Categorization Engine (core logic)
Engine contract

Implement a function:
classifyTransactions(transactions[]) -> transactions[] updated with newLabel/confidence/reason/ruleId/reviewStatus

Pattern rules (implement now)

Rule AAVE_BORROW_V1

If tx receipt logs include Aave Pool Borrow event:

newLabel = “Loan (Borrow)”

confidence = 0.95

reason = “Detected Aave Pool Borrow event”

ruleId = “AAVE_BORROW_V1”

reviewStatus = “Auto”

Rule AAVE_REPAY_V1

If logs include Repay event:

newLabel = “Loan Repayment”

confidence = 0.95

reason = “Detected Aave Pool Repay event”

ruleId = “AAVE_REPAY_V1”

reviewStatus = “Auto”

Fallback

If no match:

newLabel = originalLabel OR “Needs Review”

confidence = 0.20

reason = “No protocol rule matched”

ruleId = “NO_MATCH”

reviewStatus = “NeedsReview”

On-chain decoding requirements

Use ethers provider with RPC_URL

For each txHash:

getTransactionReceipt(txHash)

Iterate logs

Only attempt to decode logs where log.address === AAVE_POOL_ADDRESS (fast + accurate)

Use Interface with ABI containing the Borrow and Repay events only

Handle errors gracefully (rate limits, missing receipts):

mark as NeedsReview with reason “Receipt fetch failed”
